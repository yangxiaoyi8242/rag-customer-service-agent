## 问题分析
当用户输入"你好，有人在吗？"时，系统返回了"抱歉，我无法回答这个问题，请转人工客服处理。"，但没有显示询问用户是否需要调用豆包大模型的按钮。这表明交互式询问功能没有正常工作。

## 原因
系统的返回类型不正确。当用户输入的问题在资料中没有相关信息时，系统应该返回"type": "no_info"的回答，这样前端才会显示询问按钮。但现在系统返回的是"type": "info"的回答，所以前端没有显示询问按钮。

## 解决方案
修改`rag_core.py`文件中的`process_query`方法，使其在豆包返回"抱歉，我无法回答这个问题，请转人工客服处理。"时，也将回答类型修改为no_info。这样，无论relevant_docs的数量是多少，只要豆包返回"抱歉，我无法回答这个问题，请转人工客服处理。"，系统就会将回答类型修改为no_info，前端就会显示询问用户是否需要调用豆包大模型的按钮。

## 具体修改
```python
def process_query(self, query):
    """处理用户查询的完整流程"""
    # 检索相关文档
    relevant_docs = self.retrieve_relevant_docs(query)
    
    # 生成回答
    answer = self.generate_answer(query, relevant_docs)
    
    # 如果资料中没有相关信息，修改回答类型为no_info
    if len(relevant_docs) == 0 or answer.get("message") == "抱歉，我无法回答这个问题，请转人工客服处理。":
        answer = {"type": "no_info", "message": "抱歉，我无法回答这个问题，请转人工客服处理。"}
    
    return answer, relevant_docs
```
